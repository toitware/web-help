floitsch 03/15/2023 10:17 AM
Found it and fixed it.
It was a misplaced IRAM_ATTR on a variable. The C3 has a memory protection enabled that disallows writing into IRAM once the program has been loaded. It's a security feature.
However, by adding an IRAM_ATTR to one of our global variables, we crashed the system when writing into that field.
Unfortunately this happened when setting up the GPIO pins.

The fix should be included in Toit v2.0.0-alpha.66
